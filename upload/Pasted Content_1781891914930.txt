Madrasa Management ERP — Worklog
Project Overview
Building a next-generation, modular, multi-tenant Madrasa Management Ecosystem (ERP) covering:

Dynamic Feature Toggling & Multi-tenant architecture
Dynamic RBAC
Hifz Tracking Engine (Sabak / Sabaq Para / Dhor)
Shariah-compliant Finance (Zakat Tamlik automation, Funds separation)
Academic Management (Qawmi BEFAQ + Alia BMEB)
Residential / Boarding / Mess Management
HR, Payroll & Communication
Muhasaba (Tarbiyah) Tracker
AI Assistant (LLM) + Predictive Analysis
Website CMS Builder preview
Tech Stack
Next.js 16 (App Router) + TypeScript + Tailwind CSS 4 + shadcn/ui + Prisma (SQLite) + z-ai-web-dev-sdk (LLM).

Design Direction
Islamic-inspired premium theme: deep emerald/teal + gold/amber accents, dark sidebar, light content, elegant Arabic/Bengali-aware typography. No indigo/blue.

Public Action Exception Handling
Three endpoints accept public action submissions without authentication. The public branch is executed BEFORE the requireAdminAction call, returning early — admin-only branches then fall through to the permission check:

/api/website POST action="donate" — anyone can submit a donation; creates Donor (or reuses), Transaction, Fund/Account balance update, and Donation record. After the donate branch returns, the requireAdminAction(req, "WEBSITE", "EDIT") check protects create_page and other admin actions.
/api/admission POST — anyone can submit a new admission application. The PATCH handler (which approves/rejects/enrolls) is protected by ADMISSION:EDIT.
/api/feedback POST — anyone can submit a new feedback entry. The PATCH handler (assign/resolve/status update) is protected by FEEDBACK:EDIT.
