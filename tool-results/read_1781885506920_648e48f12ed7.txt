     1→# Madrasa Management ERP — Project Worklog
     2→
     3→## Project Overview
     4→A production-ready, multi-tenant SaaS Madrasa Management ERP built with Next.js 16.
     5→Designed for deployment on Coolify. Each madrasa (tenant) gets isolated data via row-level
     6→tenancy, with full RBAC, multi-language support (Bangla, English, Arabic), and a complete
     7→feature set covering academics, Hifz tracking, Shariah-compliant finance, and more.
     8→
     9→## Tech Stack (Adapted from agent's recommendation)
    10→- **Framework**: Next.js 16 with App Router (single-page SPA at `/`)
    11→- **Language**: TypeScript 5
    12→- **Styling**: Tailwind CSS 4 + shadcn/ui (New York)
    13→- **Database**: Prisma ORM + SQLite (multi-tenant via `tenantId` row-level isolation)
    14→- **Auth**: NextAuth.js v4 (credentials provider, phone-based)
    15→- **i18n**: next-intl (Bangla, English, Arabic with RTL support)
    16→- **State**: Zustand (client), TanStack Query (server)
    17→- **Charts**: Recharts
    18→- **Icons**: Lucide React
    19→
    20→## Architectural Decisions
    21→1. **Single-page app at `/`**: All user-facing UI is on `/`. Navigation handled via
    22→   Zustand store (no Next.js routing for views). API routes under `/api/*`.
    23→2. **Multi-tenancy via Row-Level Isolation**: Every domain table has `tenantId`.
    24→   A Prisma client extension auto-injects `where: { tenantId }` for tenant queries.
    25→3. **Modular folder structure**:
    26→   - `src/modules/<domain>/` — feature modules (each with api, components, store)
    27→   - `src/components/` — shared UI components
    28→   - `src/lib/` — core utilities (db, auth, i18n, tenant)
    29→   - `src/i18n/` — translations (bn, en, ar)
    30→4. **File size limit**: No file over 300 lines. Split into focused modules.
    31→5. **Three languages** with full UI translation + RTL for Arabic.
    32→
    33→## Feature Modules
    34→1. **Landing Page** — SaaS marketing, login, tenant signup
    35→2. **Dashboard** — Analytics, charts, recent activity, quick stats
    36→3. **Students** — CRUD, profiles, guardian info, class assignment, wallet
    37→4. **Teachers/Staff** — CRUD, role assignment, payroll
    38→5. **Academic** — Classes, sections, subjects, Qawmi/Alia curriculum
    39→6. **Hifz Tracking** — Sabak, Sabaq Para, Dhor with quality ratings + reports
    40→7. **Finance** — Funds (General, Lillah, Waqf, Zakat), transactions, Tamlik automation
    41→8. **Wallet** — Cashless campus, top-up, internal payments
    42→9. **Attendance** — Student & staff attendance
    43→10. **Exams & Results** — Marks entry, report cards
    44→11. **Notices** — Announcements, notifications
    45→12. **Settings** — Tenant config, language, theme, RBAC roles
    46→13. **Audit Log** — Track all critical changes
