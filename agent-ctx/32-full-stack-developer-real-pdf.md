# Task 32 — Real PDF Generation (pdf-lib)

## What was built

Replaced the fake `window.print()` PDF export approach with real downloadable PDF files generated via `pdf-lib@1.17.1`.

## Files Created
- `src/lib/pdf.ts` (202/250 lines) — pdf-lib helper library:
  - `createPdfDoc(title, tenantName)` — creates PDFDocument, adds header (top emerald bar + tenant name + title pill + date + "Generated by Madrasa Manager ERP" + divider), returns `{ doc, page, font, bold, y }`.
  - `addSection(ctx, title)` — emerald vertical accent bar + bold heading.
  - `addTable(ctx, headers, rows, colWidths)` — emerald header row + alternating emerald-tinted row backgrounds + outer border + truncate-to-fit binary search.
  - `addKpiRow(ctx, kpis[])` — N cards with emerald left accent, label/value, auto-shrink value font.
  - `addParagraph(ctx, text)` — word-wrapped paragraph with auto-pagination.
  - `addFooter(ctx, pageNum)` — "Madrasa Manager ERP" left + "Page X of Y" right + divider line.
  - `finalizePdf(ctx)` — adds footers to all pages, returns `Uint8Array`.
  - A4 (595×842pt), 50pt margins, Helvetica + HelveticaBold. Emerald accent RGB(16,185,129).
- `src/app/api/reports/pdf/route.ts` (200/200 lines) — POST endpoint:
  - Body `{ reportType }`, validates against 5 enum values.
  - Builds real PDF for: student-directory, fee-ledger, hifz-progress, finance-summary, attendance-summary.
  - Returns `Content-Type: application/pdf` + `Content-Disposition: attachment`.
  - All DB queries filter by `session.tenantId`.
- `src/app/api/export/fee-receipt-pdf/[collectionId]/route.ts` (143/150 lines) — GET endpoint:
  - Loads FeeCollection (tenant-scoped) with student + class + feeStructure.
  - Renders styled receipt: emerald "M" logo circle, "FEE RECEIPT" heading, receipt # + date, contact line, color-coded status badge (PAID/PARTIAL/PENDING/OVERDUE), student detail grid, highlighted amount box, signature line, "Jazakallahu Khairan" footer.

## Key Decisions
1. **`color` is not exported by pdf-lib** — only `rgb`, `grayscale`, `cmyk`. Initial code imported `color` which caused runtime `Export color doesn't exist in target module` errors. Fixed by replacing all `color(1,1,1)` with `rgb(1,1,1)`.
2. **StandardFonts Latin-only limitation** — all currency formatted as `BDT X` (not ৳). All dates use `en-GB` locale. Web UI continues showing Bengali/Arabic via i18n; only PDF output is LTR Latin.
3. **Auto-pagination** — `newPage()` helper invoked whenever y < MARGIN + required_height in each builder. `finalizePdf` adds footers at end (when total page count is known).
4. **Truncate-to-fit** — tables use binary search to truncate long cell text with "…" suffix based on font width measurement, preventing layout overflow.
5. **Tab→reportType mapping** — UI tab keys (students/finance/hifz/attendance/fees) differ from API reportType identifiers (student-directory/finance-summary/hifz-progress/attendance-summary/fee-ledger). A `REPORT_TYPE` constant maps between them.
6. **Existing HTML fee receipt endpoint preserved** — `/api/export/fee-receipt/[id]` is untouched (kept for backward compatibility); the new `/api/export/fee-receipt-pdf/[id]` produces a real binary PDF.
